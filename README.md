# Tetromino

A classic "falling block" puzzle game built entirely with **Jetpack Compose**.

This project was born out of a desire to swap mindless doomscrolling for mindful coding.
It serves as a personal playground for exploring reactive UI patterns, game loops in a functional UI
framework, and the occasional architectural experiment.

---

## 🚀 Purpose

The primary goal of this project is **solely educational**. It’s a "pet project" in the truest
sense:

* **Compose Exploration:** Implementation of game grids, animations, and state hoisting using modern
  Android toolkits.
* **No Pressure:** It’s not meant for production; it’s meant for learning.

## 🛠 Tech Stack

* **Language:** [Kotlin](https://kotlinlang.org/)
* **UI Framework:** [Jetpack Compose](https://developer.android.com/jetpack/compose)
* **State Management:** ViewModel, StateFlow, and Compose State
* **Concurrency:** Kotlin Coroutines for the game loop timing

## ⚠️ The "Pet Project" Disclaimer

Since this project was built for fun and learning rather than a commercial release, please note:

* **Bugs are likely:**
* **Questionable Decisions:** Some architectural choices were made to test a theory or simply
  because it was late at night. It is not always "Clean Code™" approved — it's "Works on my Device™"
  approved.
* **Testing:** Manual testing was performed by playing the game. Formal unit tests may be sparse or
  non-existent.

> "If the code is messy but the blocks still disappear when they form a line, it's a win."
